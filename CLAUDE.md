# CLAUDE.md

## 작업 방식

- 코드 변경 전 항상 플랜 모드로 계획을 먼저 제시하고 승인 후 적용

## 코드 스타일

- 함수는 항상 arrow function으로 작성

```js
// ✅
export const foo = () => {};

// ❌
export function foo() {}
```

## 설계 원칙

**사용자 범용성 최우선**

이 라이브러리의 사용자는 다양한 환경과 코드 스타일을 가진다.

- **모듈 시스템**: ESM(`import`)과 CJS(`require`) 모두 지원
- **코드 스타일**: 어떤 방식으로 작성된 코드도 모킹·테스트 가능
- **환경**: 다양한 Node.js 환경과 프로젝트 구조에서 동작

새 기능 설계 시 "다양한 사용자 환경에서도 동작하는가?" 를 가장 먼저 고려한다.
